数据库系统表

  • 上图，第一行的数据库，为系统自带数据库，包含了数据库中所有表和列的信息，

    其中SCHEMATA表中记录了有关数据库的信息

    TABLES表中记录了有关表的信息

    COLUMNS表中记录了有关字段的信息

  • 获取数据的相关语句

    1. 查询用户数据库相关信息

      select SCHEMA_NAME from INFORMATION_SCHEMA.SCHEMATA;

      从上述图中可以看到在schemata中有一个名为schema_name的字段，该字段中储存着数据库名，所以上述语句意为查询出所有的数据库名

    2. 查询数据表相关信息

      在上一个步骤中，我们查到了数据库的相关信息，大部分情况下，我们是要利用上一步查到的信息来对接下来的查询做一个限制

      select TABLE_NAME from INFORMATION_SCHEMA.TABLES where TABLE_SCHEMA =(select DATABASE() )

      上述语句的意思为查询当前数据库中数据表名，当然你也可以不查询当前数据库的，这是一个需要随机应变的点。

    3. 查询字段的相关信息

      select COLUMN_NAME from INFORMATION_SCHEMA.COLUMNS where TABLE_NAME=”要查的字段所属的表名“

联合注入过滤绕过技巧

联合注入不适用情形：

1.union关键字被完全过滤

2.页面中压根不返回查询数据

  1. php中常用替换函数

    str_replace

    preg_replace

    preg_match

  2. 大小写绕过

    基础：在Mysql中，大小写字母的含义是一致的。如果在进行过滤提交的数据过程中，没有对大小写进行区分，那么此时就会造成大小写绕过过滤的情况

3.双写绕过

基础;在使用preg_replace函数过程中，默认情况下只进行一次匹配，因此 如果匹配到字符替换为空的情况，就可以造成双写绕过

4.过滤单引号绕过 十六进制

基础：在mysql数据库中的sql语句，对于字符串数据必须使用引号。但是对于字符串来说，mysql也识别字符串中每个字符对应的ASCII码的16进制，此时可以使用0x16进制替换字符串，从而绕过引号对字符串的限制。

  1. 宽字节注入原理与利用

    基础：如果数据库中存储数据使用的编码方式是GBK，那么由于用户输入的内容会进行双字节的组合，会导致用户输入的字节与反斜杠组合，从而逃逸引号.
